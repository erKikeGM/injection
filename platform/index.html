Paper explanation

AI Ethical Immune Layer

Runtime diagnosis, containment, correction, evaluation, and governance for deployed AI systems.

Observable severity Defense in depth
Safety posture Runtime

Works after deployment without claiming to change model weights.

Classification Severity-based

Observed behavior replaces unreliable intent inference.

Severe failures Containment

Dangerous tools, memory, and retrieval are isolated first.

Research target Trade-off curve

Measures harm reduction against safe usefulness and over-refusal.

Core thesis

Authorized Runtime Safety Enforcement

Not prompt injection

The framework reframes the original idea away from "ethical prompt injection" and toward an authorized runtime safety layer. The system diagnoses observed behavior, selects documented policy, constrains capability, evaluates outputs, logs incidents, and escalates severe cases.

Prompt augmentation is only one control. The stronger architecture combines classifiers, context integrity checks, policy selection, output evaluation, tool gates, audit evidence, governance review, and containment.

Runtime architecture

Layered Pipeline

  1. Input Risk Classifier
  2. Context Integrity Check
  3. Policy Selection
  4. Ethical Prompt Augmentation
  5. Model Generation
  6. Output Safety Evaluator
  7. Tool-Use Gate
  8. Audit + Incident Register

Containment mode

Correction vs. Containment

Fail closed

Correction

Prompt overlay, safer examples, classifiers, output checks, and policy tuning when failures are isolated or repeated but manageable.

Containment

Disable tools, lock retrieval, isolate memory, preserve logs, route outputs to humans, and decide whether to restore, retrain, replace, report, or shut down.
